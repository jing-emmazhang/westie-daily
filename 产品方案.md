# 西高地每日热点 · 产品方案与策略

## 产品定位

帮助小红书宠物博主（养了一只西高地白梗）每天从 Instagram 发现真正新火的狗狗内容，自动生成西高地版本的生图提示词，经用户确认后发布——**用更少精力做出更有质量的内容**。

目标不是批量产内容，而是每天最多一条、但每条都值得发。

---

## 用户痛点

- 每天手动刷 Instagram 找灵感，耗时且低效
- 想跟热点，但发出去的内容要有自己的狗狗风格，不能直接搬运
- 不想失去对最终内容的控制权（选哪条、发不发）
- 整个流程尽量不需要打开电脑操作

---

## MVP 工作流

```
每天 10:30（台湾时间）
  └→ 自动抓取 Instagram 热帖（5个话题 + 3个账号）
  └→ 评分取 Top 10，生成西高地生图提示词
  └→ 推送到 GitHub Pages，页面自动更新

用户打开 https://jing-emmazhang.github.io/westie-daily/
  └→ 浏览 Top 10 卡片，查看原图、赞数、评分明细
  └→ 复制中意的生图提示词 → 去豆包 / Midjourney 生图
  └→ 标记"已采用"，记录本次选择
  └→ 配文案发小红书
```

## 内容抓取策略

### 来源构成

| 来源 | 说明 | 来源加成 |
|------|------|---------|
| Instagram 发现页 | 搜索 `dog`（不带#），平台算法主动推荐，背书最强 | +2 |
| 关键词话题页 | `#dog` `#puppy` `#dogsofinstagram` | +0 |
| 固定账号 | `jordhammond` `tom_juenemann`（自然风光，适合作西高地背景） | +1 |

风景账号（jordhammond / tom_juenemann）保留原因：富士山、田野、雪景等场景在社交媒体上极火，可直接作为西高地照片的背景灵感来源。

已移除：`#dogoftheday` `#fluffydogs`（个人随手打标签，赞数普遍 <100）；`crownandpaw`（商业账号，点赞极低）。

### 评分规则（满分 7 分）

| 维度 | 分值 | 说明 |
|------|------|------|
| 绝对热度 | 0–4 分 | 对数打分：1k赞≈1分，10k≈2.5分，100k=4分 |
| 来源加成 | 0–2 分 | 发现页 +2，账号 +1，话题页 +0 |
| 时效性 | 0–1 分 | 10天内发布得1分 |
| 同分排序 | — | 赞数高者优先（避免高赞帖因插入顺序靠后而沉底）|

**设计原则**：不引入互动率维度（赞数/粉丝数），避免小号偶发爆帖干扰排名；绝对热度是最可靠的内容质量代理指标。

## 西高地生图提示词策略

每条帖子自动生成一段英文提示词，模板为：

```
A West Highland White Terrier, pure white fluffy fur, bright eyes, small black nose.
Scene inspired by: {帖子亮点或描述}
Natural lighting, photorealistic, square composition, no text, no watermark.
```

- 以帖子的亮点（highlight）或描述作为场景输入
- 生成结果固定以西高地为主体，Instagram 原帖只作场景灵感
- 推荐配合豆包 Seedream（支持中文指令 + img2img + 2K分辨率）

## 页面功能设计

| 功能 | 说明 |
|------|------|
| Top 10 卡片网格 | 展示排名、来源标签、原图、赞数、得分 |
| 查看原帖 | 直接跳转 Instagram 原帖 |
| 复制图片 | 一键复制图片到剪贴板（跨域降级为新标签打开）|
| 复制提示词 | 一键复制西高地生图 prompt |
| 评分明细 | 可展开查看各维度分值条形图 |
| 标记采用 | 本地记录已选用的帖子，卡片变灰避免重复 |
| 历史日期 | 下拉选择往期数据，可回溯每日 Top 10 |

## 技术架构

```
本地脚本（Claude in Chrome）
  ↓ 抓取 Instagram，提取 like_count / taken_at / display_url
  ↓ 调用 scrape_mvp.py → rank_and_save()
  ↓ 写入 westie-daily/mvp/data/top10_YYYY-MM-DD.json
  ↓ auto_push.sh → git commit + push

GitHub Pages（静态托管）
  ↓ index.html fetch mvp/data/index.json + top10_DATE.json
  ↓ 纯前端渲染，无后端，永久免费
```

**状态文件**（均为人类可读 JSON，出错易排查）：

| 文件 | 用途 |
|------|------|
| `mvp/data/top10_YYYY-MM-DD.json` | 每日 Top 10 结果 |
| `mvp/data/index.json` | 所有历史日期索引 |

## 运行方式

- **自动**：每天台湾时间 10:30，Cowork 定时任务自动抓取 + 推送，页面静默更新
- **手动触发**：在 Cowork 侧边栏找到"westie-daily-scrape"点击 Run now
- **查看页面**：https://jing-emmazhang.github.io/westie-daily/

> 定时任务需要 Cowork 保持打开状态；若当天 App 关闭，下次启动时自动补跑。

## 后续迭代方向

1. **邮件选帖**：每天抓取后发邮件给自己（Gmail），回复选出想生图的帖子，实现更轻量的移动端审核
2. **豆包生图集成**：对选中帖子自动调用豆包 Seedream API，生成西高地版图片，减少手动操作
3. **小红书发布**：Claude in Chrome 打开小红书，自动粘贴图文，等用户确认后发布
4. **去重记忆**：引入 `seen_posts.json` 记录历史推送，同一帖子最多推送 3 次后不再出现
5. **文案自动生成**：结合帖子内容和当前节气/热点，自动生成适配小红书风格的中文文案
